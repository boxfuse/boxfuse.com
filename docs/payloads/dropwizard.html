Dropwizard

Boxfuse supports Dropwizard 0.8.x or newer Apps packaged as a Dropwizard Executable Jar using either OpenJDK 7.x or 8.x.

Configuration

Boxfuse extracts all .yml files at the root of the classpath to the same directory as the executable jar.

By default Boxfuse runs your Dropwizard app with the arguments server boxfuse.yml.

If no boxfuse.yml is found at the root of the classpath, Boxfuse uses the following defaults:

server:
  applicationConnectors:
    - type: http
      port: 80

The use a different yml config file, or a different set of arguments, set jvm.main.args with the arguments you need.

Healthchecks

Unless specified otherwise, Boxfuse automatically configures healthcheck.port=8081 and healthcheck.path=/healthcheckto match Dropwizard's defaults.

Root Certificates

By default, Boxfuse uses the same root certificate bundle as the latest version of Firefox. You can, however, ship your own set of root certificates, by placing them in a KeyStore inside the Jar file as /cacerts. Boxfuse will then automatically configure the JRE to use these instead.

Java Agents

If you wish to launch the JRE with one or more Java Agents, simply place the Java Agent Jar files inside the Jar file under /javaagents/. Boxfuse will then automatically configure the JRE to use these Java Agents.
