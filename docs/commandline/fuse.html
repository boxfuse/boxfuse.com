fuse

Fuses a Payload together with the Components it requires into an Image.

Usage: boxfuse fuse payload

If no Payload is specified, Boxfuse will search for a suitable one in the current directory.

> boxfuse fuse hello-1.0.war

Fusing Image for hello-1.0.war ...
Image fused in 00:07.351s (48147 K) -> axelfontaine/hello:1.0

Supported Payload formats:

  • Executable jar file
  • War file
  • Play dist zip

Arguments

Name Default Description
-components.NAME=version latest version Pins the component with this name onto this version.
Example: -components.tomcat=8.0.9
-debug false Start the Payload in debug mode when an Instance launches
-debug.wait false Whether the JVM should wait for the remote debugger to connect (Only applicable when debug mode is active)
-payload.port autodetected The main port on which the payload starts (usually http or https)
-payload.path / The path where the payload starts
-healthcheck true Check whether to payload started correctly
-healthcheck.port=... autodetected The port to check whether to payload started correctly
-healthcheck.path=... / The path to check whether to payload started correctly
-healthcheck.timeout=... 60 The number of seconds to wait for the Payload to come up
-jvm.args=... Extra arguments to pass to the JVM
-jvm.jmx false Enable the JMX remote management and profiling interface for the JVM
-jvm.main.class=... autodetected Main class to invoke on JVM startup
-jvm.main.args=... Arguments to pass to the main class
-ports.NAME=definition http=80

Exposes the port of the app with this name using this definition. Example: -ports.jmx=8001

Supported formats:

80TCP port 80, universally accessible
80/tcpTCP port 80, universally accessible
80/udpUDP port 80, universally accessible
80/tcp:@TCP port 80 only accessible from your own IP
80/udp:@/20UDP port 80 only accessible from the IPs in the CIDR /20 block of your own IP
80/udp:1.2.3.4UDP port 80 only accessible from 1.2.3.4
80/tcp:1.2.3.4/31TCP port 80 only accessible from the IPs in the CIDR /31 block of 1.2.3.4
-tmp=... 1 The amount of temp space to allocate to /tmp in GB
-vault false Immediately upload the new Image to the Boxfuse Vault
