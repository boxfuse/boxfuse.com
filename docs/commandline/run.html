run

Runs an Instance of this Image on your machine or on AWS.

Usage: boxfuse run payload | image

> boxfuse run hello:1.0

Launching Instance of axelfontaine/hello:1.0 on VirtualBox ...
Forwarding http port localhost:8888 -> vb-ec717c5e:80
Instance launched in 00:04.006s -> vb-ec717c5e
Waiting for Payload to start on Instance vb-ec717c5e ...
Payload started in 00:08.786s -> http://127.0.0.1:8888

Arguments:

Name Default Description
-cpus=... same as host The number of CPUs to assign to an Instance (VirtualBox only)
-env=prod dev Deploys on AWS instead of on your local machine
-envvars.NAME=value Passes this environment variable into the Instance
Example: -env.JDBC_URL=jdbc:mydburl
-healtcheck true Check whether to payload started correctly
-healtcheck.port=... autodetected The port to check whether to payload started correctly
-healtcheck.path=... / The path to check whether to payload started correctly
-healtcheck.timeout=... 60 The number of seconds to wait for the Payload to come up
-jvm.args=... Extra arguments to pass to the JVM
-jvm.main.class=... autodetected Main class to invoke on JVM startup
-jvm.main.args=... Arguments to pass to the main class
-logs true Logs may slow down the Instance. Set to false to disable. (VirtualBox on Windows only)
-logs.auto true Whether to automatically display the instance logs on startup
-logs.dir=... The directory where the logs should be redirected to
-ports.NAME=number http=80 Maps the port of the app with this name onto this number.
Example: -ports.jmx=8001
-portsmap.NAME=number http=8888 The local ports to forward to the running Instance (VirtualBox only)
Example: -portsmap.jmx=8111
(if ports.jmx=8001 this will forward localhost:8111 to instance:8001)
-ram=... 1024 The amount of RAM in MB to assign to an Instance (VirtualBox only)
