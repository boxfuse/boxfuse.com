Terminology

Before we can start we have to talk about a few simple terms you will encounter:

Boxfuse Dashboard: The online dashboard to manage your Boxfuse account and usage.

Boxfuse Client: The installable client that lets you invoke the various commands pictured above.

Boxfuse Component Inventory: The online inventory of all available versions of the various components used to fuse a Bootable App.

Component: A dependency of your app provided by the Boxfuse Component Inventory. Examples: Tomcat, OpenJDK, ...

App: This is your application: the payload of the Bootable App. Boxfuse supports the usual standard formats like executable .jar and .war files, avoiding any lock in.

Bootable App: The universally deployable immutable unit containing both your app and all the components it needs to run.

Platform: An execution platform for a Bootable App. Examples: VirtualBox, AWS, ...

Image: A Bootable App converted to a format supported by a specific platform. Examples: VMDK, AMI, ...

Instance: A running instance of an Image on a specific platform. Examples: VirtualBox VM, AWS Instance, ...

Details

Now that we've covered some basic terms, let's look at what the commands of the Boxfuse Client do:

versions: Lists all versions of a specific Component available in the Boxfuse Component Inventory. For Tomcat this could be 7.0.54, 8.0.8, 8.0.9, ...

fuse: Fuses your App together with the Components it requires into a Bootable App.

ls: Lists all available Bootable Apps.

rm: Removes a Bootable App.

convert: Converts a Bootable App into a Platform-specific Image.

lsi: Lists all available Images.

rmi: Removes an Image.

run: Runs an Instance of an Image on a specific Platform.

ps: Lists all running Instances.

kill: Gracefully kill an Instance.

open: Opens a web browser pointing at the https or http port of an Instance.

logs: Display the console logs of an Instance.
